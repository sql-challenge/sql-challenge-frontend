# SQL Challenger

## Product Purpose
SQL Challenger é uma plataforma gamificada de aprendizado de SQL com tema detetivesco. Usuários resolvem mistérios escrevendo queries SQL reais em um editor interativo, com progressão por capítulos, sistema de XP, conquistas, ranking, e dicas com penalidade.

## Users
- **Público-alvo**: Desenvolvedores iniciantes e intermediários (18-35 anos) que querem aprender SQL de forma prática e envolvente
- **Contexto de uso**: Sessões de estudo autônomo de 15-60 minutos, em desktop (primário) ou mobile (secundário), geralmente à noite após o trabalho/faculdade
- **Estado de espírito**: Curiosos, mas cansados de cursos tradicionais. Buscam algo que pareça um jogo, não uma aula.
- **Frequência**: 2-5x por semana, em sprints de aprendizado

## Brand & Tone
- **Registro**: product (app UI com momentos brand para landing page)
- **Personalidade**: Investigação noir + fantasia urbana. O usuário é um detetive SQL em um mundo mágico.
- **Tom**: Confiante, misterioso, recompensador. Nunca infantil. Nunca corporativo.
- **Voz**: "Você está prestes a desvendar um caso. Os dados estão esperando. O que você encontra?"

## Design Direction
- **Estratégia de cor**: Restrained na base do app (product), com momentos de Committed em landing e conquistas
- **Tema**: Dark mode como default (detetive trabalha à noite em salas mal iluminadas), light mode como alternativo
- **Vibes de referência**:
  - LA Noire / disco azul escuro + âmbar
  - Sherlock (BBC) / âmbar + azul marinho
  - Disco Elysium / tons sujos, UI diegética
  - Linear / clareza de product UI

## Anti-References
Evitar absolutamente:
- Gradientes roxo-rosa (AI slop)
- Cards de vidro/glassmorphism sem propósito
- Ilustrações genéricas de vetor
- Tipografia Inter/Roboto/Arial
- Cores pastéis ou "saas-cream"
- Emoticons genéricos trocados por icons consistentes
- Layouts centrados-simétricos sem variação

## Strategic Principles
1. **Detetive primeiro, app segundo** — cada tela deve parecer uma ferramenta de investigação, não um dashboard SaaS
2. **Progresso tangível** — XP, níveis, conquistas precisam ser celebrados com micro-interações animadas
3. **Foco na prática** — o editor SQL é o centro; a UI ao redor apoia sem roubar a cena
4. **Consistência é affordance** — mesmos patterns de componente em toda parte (product register)
5. **Delight nos momentos certos** — celebração nas conclusões, não nos cliques diários
